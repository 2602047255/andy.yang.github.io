webpack 基本概述

webpack 本质是一个第三方模块包，用于分析并打包代码

  • 支持所有类型文件的打包
  • 支持 less/sass 转换为 css
  • 支持 ES6/7/8 转换为 ES5
  • 压缩代码，提高加载速度

webpack 使用步骤

  • 初始化包环境：yarn init
  • 安装依赖包：yarn add webpack webpack-cli -D
  • 配置 package.json 的 scripts(自定义命令) 节点 ："scripts": { "build": "webpack" }

webpack 基础使用

目录结构（以下文件由自己手动创建）：

project

| — src

| —— add

| ———— add.js

| —— css

| ———— index.css

| —— assets

| —— index.html

| —— index.js (webpack 打包的入口)

  • add.js - 定义求和函数并导出
  • index.js - 导入 add 模块并使用求和函数输出结果
  • 项目根目录执行 yarn build 自定义命令，进行打包

入口和出口

webpack 默认的入口是 src/index.js，默认的出口是 dist/main.js，这些配置信息都可以通过 webpack.config.js 来进行修改，webpack.config.js 需要新建在项目的根目录

注意：修改了入口的路径之后，目录需要与入口路径一 一对应

隔行变色案例

当 main.js 的代码写完后执行 yarn build 进行打包，然后将 src/index.html 复制到 build/index.html，然后在 build/index.html 引入 bundle.js，最后在浏览器上打开 build/index.html

生成 HTML 文件

HTMLWebpackPlugin 可以简化 HTM 文件的创建，以便为 webpack 包提供服务，该插件会自动生成一个 HTML 文件，当然，你需要通过 yarn add html-webpack-plugin -D 命令进行安装

打包 CSS 文件

打包 CSS 文件时，需要用到 css-loader 和 style-loader，这两个模块，其中 css-loader 是为了让 webpack 能处理 css 类型文件，style-loader 用于把 css 插入到 DOM 中，你可以通过 yarn add css-loader style-loader -D 进行安装

打包完之后，在浏览器的控制台中，会发现多了一个 style 标签，里面就有你所写的 css 代码

打包 Less 文件

打包 Less 文件时，需要用到 less 和 less-loader，这两个模块，其中 less-loader 是为了让 webpack 能处理 less 类型文件，less 用于把 less 编译为 css，你可以通过 yarn add less less-loader -D 进行安装

处理图片文件

小于 8KB 转为 base64 的好处就是浏览器不用发请求，可以直接读取，速度快。相反，如果大于 8KB 的图片转为 base64 就会让项目体积增大 30% 左右

处理浏览器兼容问题

当一些高级的 JavaScript 语法对低版本的浏览器不兼容时，babel 这个模块就可以进行兼容的处理，可以通过 yarn add babel-loader @babel/core @babel/preset-env -D 进行安装

自动打包工具

问题：每次修改代码之后，都要重新 yarn build 打包，才能看到最新的效果，实际工作中，打包 yarn build 是非常费时的，大约 30s~60s

  • 原因：
    • 从 0 开始构建依赖
    • 磁盘读取对应的文件到内存，webpack 开始加载
    • 再用对应的 loader 进行处理
    • 将处理完的内容，输出到磁盘指定的目录
  • 解决：
    • 启动一个开发服务器
    • 缓存一些已经打包过的内容，只重新打包修改过的文件
    • 最终运行在内存中给浏览器使用
  • 总结：
    • 把代码运行在内存中，自动更新，实时返回给浏览器显示

webpack 的自动打包是需要下载第三方模块的，操作步骤如下：

  • 下载模块包
    • yarn add webpack-dev-server -D
  • 在 package.json 里面配置自定义服务器启动命令
    • "scripts": { "serve": "webpack serve" }
  • 启动服务器
    • yarn serve
    • 浏览器上访问 localhost:8080

自动打包的配置可以通过 wepback.config.js 进行自定义的配置，比如：修改端口、自动打开浏览器等

更多配置可以在官方文档上查阅：https://webpack.docschina.org/concepts/
